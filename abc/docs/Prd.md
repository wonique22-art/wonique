# 팀 랜덤 선택기 웹앱 - PRD

## 1. 개요 (Overview)

팀원들이 회식 메뉴 선택, 발표 순서 정하기 등 의사결정이 필요한 상황에서 **공정하고 재미있는** 랜덤 선택 기능을 제공하는 웹앱입니다. 모바일에서도 사용 가능해야 합니다.


## 2. 목적 (Purpose)

- 팀 의사결정 과정을 간단하고 재미있게 진행
- 공정하고 투명한 선택 방식 제공
- 회의/회식 등에서 빠른 결정 지원

## 3. 주요 기능 (Features)

### 3.1 입력 방식

#### 1) URL 파라미터 입력
- 쿼리 파라미터로 팀원/항목 전달
- 예: `/?items=팀원1,팀원2,팀원3`
- 쉘 스크립트/즐겨찾기로 빠른 접근 가능

#### 2) 텍스트 파일 업로드
- `.txt` 파일 업로드 (라인별로 팀원/항목 구분)
- 드래그 & 드롭 지원
- 파일 내용 자동 파싱

#### 3) 직접 입력
- 텍스트 에러어 또는 텍스트 영역에서 직접 입력
- 쉼표 또는 줄바꿈으로 구분

### 3.2 선택 기능

- **랜덤 선택**: 리스트에서 하나의 항목 무작위 선택
- **셔플 / 순서 정하기**: 전체 목록을 랜덤한 순서로 재배열
- **복원력 있는 UI**: 선택된 항목 하이라이트, 순서 표시

### 3.3 애니메이션

- 선택 중: 빠른 스크롤/스핀 애니메이션으로 긴장감 연출
- 선택 완료: 부드러운 정지 및 강조 애니메이션
- 순서 정하기: 카드 섞이는 애니메이션
- 마이크로 인터랙션: 호버 효과, 깔끔한 전환 효과

## 4. 기술 스택

- **Framework**: Next.js 13+
- **CSS**: Tailwind CSS
- **애니메이션**: CSS Animations & Framer Motion (선택)
- **배포**: Vercel

## 5. 페이지 구조

### 5.1 홈 페이지 (/)
- 입력 방식 선택 UI
- 파일 업로드 영역
- 텍스트 입력 영역
- 항목 리스트 프리뷰

### 5.2 선택 페이지 (/select)
- 입력받은 항목들 표시
- 무작위 선택 버튼
- 애니메이션 효과
- 선택 결과 표시
- 처음으로 돌아가기 버튼

### 5.3 순서 정하기 페이지 (/shuffle)
- 입력받은 항목들을 셔플
- 순서가 표시된 카드 형태
- 재셔플 버튼
- 결과 내보내기 옵션

## 6. UI/UX 설계

### 디자인 원칙
- **모던함**: 클린한 라이아웃, 미니멀한 디자인
- **심플함**: 1-2 클릭으로 결과까지 도달 가능
- **직관성**: 명확한 CTA(Call To Action)

### 색상 팔레트
- Primary: 활기찬 색상 (예: 파란색, 보라색)
- Secondary: 중립적인 색상
- Background: 밝고 깨끗한 배경
- Accent: 선택 결과 강조용

### 폰트
- 한글: 명확하고 읽기 좋은 산세리프
- 모던한 폰트 weight 조합

## 7. 사용자 흐름 (User Flow)

```
홈페이지 
  ↓
입력 방식 선택 (URL / 파일 / 직접입력)
  ↓
항목 입력 & 확인
  ↓
선택 또는 셔플 선택
  ↓
애니메이션 + 결과 표시
  ↓
재선택 또는 처음으로
```

## 8. 핵심 사용 사례 (Use Cases)

1. **회식 메뉴 선택**: 팀원 리스트 → 한 명 선택 → 그 사람이 고르는 메뉴 투표
2. **발표 순서**: 팀원 리스트 → 순서 정하기 → 1번부터 발표 진행
3. **미팅 진행자**: 주제 리스트 → 랜덤 선택 → 토론할 주제 결정
4. **팀빌딩**: 게임/활동 리스트 → 순서 정하기 → 순차 진행

## 9. 데이터 처리

- 입력 데이터는 클라이언트 사이드에서만 처리
- 브라우저 로컬스토리지: 최근 사용 목록 저장 (선택사항)
- 서버 저장 없음 (개인정보 보호)

## 10. 성능 & 호환성

- 가볍고 빠른 로딩
- 모바일 반응형 디자인
- 크로스브라우저 호환성

## 11. 향후 확장 기능 (Future Enhancements)

- 선택 기록 저장 / 내보내기

- ✅ 매끄러운 애니메이션과 함께 결과 도출
- ✅ 모바일에서도 쾌적한 사용성
